## 📝 开发规范

### 分支命名规范
- 功能开发: `feature/v4.0-功能名称`
- Bug修复: `bugfix/v4.0-问题描述`
- 性能优化: `perf/v4.0-优化项`
- 文档更新: `docs/v4.0-文档说明`

### 提交信息规范
```
类型(范围): 简短描述

详细描述（可选）

Breaking Changes（如有）
```

类型：
- feat: 新功能
- fix: Bug修复
- perf: 性能优化
- refactor: 代码重构
- docs: 文档更新
- test: 测试相关
- chore: 构建/工具链相关

### 代码审查要求
- 所有功能必须有单元测试
- 代码覆盖率不低于80%
- 必须通过所有CI检查
- 至少一人审查通过
